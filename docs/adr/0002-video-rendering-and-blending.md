# ADR-0002: 映像のレンダリングと合成方式

日付: 2026-06-12
ステータス: 採用

## コンテキスト

2つのデッキの映像をフェーダーでブレンドして出力する必要がある。なるべくCPUを使わず、ハードウェアアクセラレーションを活かしたい（WANT要件）。

候補:

1. `<video>` 要素2枚を重ね、上側のCSS `opacity` をフェーダー値にする
2. Canvas 2D に毎フレーム `drawImage` して `globalAlpha` で合成
3. WebGLでテクスチャとして合成（クロスフェード以外のエフェクトも可能）

## 決定

**案1（`<video>` ×2 + CSS opacity）** を採用する。

## 理由

- `<video>` のH.264デコードはブラウザがハードウェアデコーダを使い、CSS opacityの合成はGPUコンポジタで行われるため、JavaScript側のフレーム処理コストがゼロになる
- Canvas 2D / WebGLは毎フレームのGPU→GPUコピーやJSループが必要で、最も単純な要件（クロスフェード）に対して複雑さが過剰
- 要件は「A/Bどちらを出すかをフェーダーで切り替え」であり、opacityによるクロスフェードで必要十分

## 結果

- ルマキー等の高度なミックスはできない。TODO: エフェクトが必要になったらWebGL（案3）へ移行する
- 透過合成のため、フェーダー中間位置では単純なアルファブレンドになる
