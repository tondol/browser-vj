# ADR-0001: 言語とビルドツールの選定

日付: 2026-06-12
ステータス: 採用

## コンテキスト

開発ルールとして「静的型付け言語の使用」「シンプルな選択肢の採用」「メンテが継続され広く使われたライブラリの選定」「更新直後のライブラリを避ける」が定められている。

## 決定

- 言語: **TypeScript**（strictモード）
- ビルド: **Vite**
- UIフレームワーク: **使わない**（Vanilla TS + 静的HTML）

依存はdevDependenciesの `typescript` と `vite` の2つのみとし、ランタイム依存をゼロにする。バージョンは公開から一定期間（数週間以上）経過したものに固定（`-E`）してインストールする。

## 理由

- TypeScriptはブラウザ向け静的型付け言語のデファクトであり、コンパイル時のエラー検出という要件を最小コストで満たす
- 本アプリのUIは2つのデッキ＋フェーダー＋ライブラリ程度で状態が少なく、React等のフレームワークは複雑さに見合わない
- 依存を2つに絞ることでサプライチェーンリスクを最小化できる

## 結果

- 状態管理・DOM更新はすべて手書きになるが、規模が小さいため許容範囲
