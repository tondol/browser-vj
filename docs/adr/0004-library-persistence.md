# ADR-0004: ライブラリの永続化方式

日付: 2026-06-12
ステータス: 廃止（ADR-0010 で永続化そのものを取りやめた）

## コンテキスト

毎回ファイラーで探さずにデッキへロードできるライブラリ機能（WANT）には、ブラウザ再起動後もファイルへの参照を保持する仕組みが必要。

候補:

1. File System Access APIの `FileSystemFileHandle` をIndexedDBに保存する
2. 動画ファイルの中身そのものをIndexedDB（またはOPFS）へコピーして保存する
3. 永続化せずセッション内のみのライブラリにする

## 決定

**案1（FileSystemFileHandleをIndexedDBへ保存）** を採用する。File System Access APIが使えないブラウザ（Firefox等）では自動的に案3（セッション内のみ、メモリ保持）へフォールバックし、UI上に注記を表示する。

## 理由

- ハンドル保存はファイルをコピーしないためストレージを消費せず、元ファイルの更新もそのまま反映される
- 案2はGB単位の動画でストレージを二重消費するため不採用
- File System Access APIはChromium系で安定しており、VJ用途では実用上Chromium系の利用を前提にできる

## 結果

- ブラウザ再起動後の初回ロード時に、ハンドルごとに読み取り許可のプロンプトが出る（ユーザー操作1回で許可）
- ファイルが移動・削除されるとロードに失敗するため、その旨をアラート表示する
