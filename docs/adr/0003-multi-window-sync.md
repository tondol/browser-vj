# ADR-0003: 出力ウィンドウとの同期方式

日付: 2026-06-12
ステータス: 採用

## コンテキスト

コントローラ（メインウィンドウ）とは別に、出力ウィンドウ・プレビューウィンドウへ同じ映像を表示する必要がある。

候補:

1. 各ウィンドウが自前の `<video>` を持ち、BroadcastChannelで操作と再生位置を同期する
2. `canvas.captureStream()` のMediaStreamを出力側へ転送する
3. 出力側だけが `<video>` を持ち、コントローラは映像なしで操作のみ行う

## 決定

**案1（BroadcastChannel + 各ウィンドウが自前のvideoを再生）** を採用する。

- 動画ファイルは `URL.createObjectURL()` のBlob URLとして共有する（同一オリジンのウィンドウ間で有効）
- コントローラをマスタークロックとし、500ms間隔で再生位置を `sync` メッセージ配信する
- ミラー側は ずれ>0.5s で即シーク、0.02〜0.5s は `playbackRate` の±5%調整で滑らかに追従する

## 理由

- MediaStreamのウィンドウ間転送（案2）はブラウザ対応が不安定で、canvas経由はCPU/GPUコストも増える
- 案3は出力ウィンドウを閉じると再生状態が失われ、ウィンドウの開き直しに弱い
- 案1はウィンドウをいつ開いても `hello` メッセージで現在状態を取得でき、複数プレビューも同じ仕組みで動く

## 結果

- 同一ファイルをウィンドウごとにデコードするため、ウィンドウ数に比例してデコード負荷が増える（ハードウェアデコードのため実用上は軽い）
- ウィンドウ間の同期精度はフレーム単位の保証はなく、最大±20ms程度のずれが残りうる
